__Criando uma classe de Serviço no Angular(Service)__

Fornece os dados e contém a lógica de negócios da aplicação.

Notação/Decorator de um serviço: @Injectable
-- Adiciona informações essenciais sobre o ciclo de vida da classe do Angular
-- Exemplo: OnInit

A notação Injectable insere as dependências automaticamente, além de instanciar o objeto.
Para isso, só precisamos declarar a variável e seu tipo no contrutor do Service.
Dessa forma, por exemplo
	constructor(private httpClient: HttpClient) { }

Dados e manipulação dos dados para que o componente receba os mesmos de forma correta.
Da forma que necessita para ser facilitado no componente.

Comando: ng g service [name-do-service]

Comando utilizado para criar dentro da pasta courses/services:
	ng g service courses/services/courses

Criamos um que faz uma requisição utilizando a classe do angular HttpClient.
HttpClient faz a conexão com a API.



<div *ngIf="courses$ | async as courses; else loading">


__Tratamento de Erros__
Criaremos um componente para erros.

Criaremos um outro módulo para compartilhar uma popup de erros.
Chamaremos de shared
Comando: ng g m shared

Criaremos o componente:
	ng g c shared/components/error-dialog
